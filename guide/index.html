Setup Guide

歡迎！接下來要做什麼？

感謝您購買 One Page Go 方案，以下是設定流程總覽

您的網頁需要兩個服務來運作：


  1. GitHub Pages — 網頁託管
    您的網頁會放在 GitHub 上，透過 GitHub Pages 免費提供全球訪問。
  2. Google Sheet — 資料後台
    訂單、會員、商品等資料存在您自己的 Google Sheet，完全由您掌控。
為什麼要自己建立帳號？
所有資料都存放在您自己的帳號中，即使日後不再使用我們的服務，您的資料依然屬於您，不會遺失或被刪除。

預估所需時間
整個流程約需 10-15 分鐘。如果您已經有 Google 帳號，會更快。

Step 1：建立 GitHub 帳號

如果您已有 GitHub 帳號，可跳到 Step 2

  1. 前往 github.com，點擊右上角 Sign up
  2. 輸入您的 Email 地址，設定密碼
  3. 選擇一個 使用者名稱（英文 + 數字，不含空格）
    例如：my-bakery-shop、wang-studio
  4. 完成驗證後，到信箱點擊 驗證連結 啟用帳號
使用者名稱建議
GitHub 使用者名稱會出現在您的網址中（例如 your-name.github.io），建議使用品牌相關的簡短英文名稱。
免費方案即可
GitHub 免費方案已包含 GitHub Pages 功能，不需要付費升級。

Step 2：建立 Repository

Repository（倉庫）是存放網頁檔案的地方

  1. 登入 GitHub 後，點擊右上角 + 選擇 New repository
  2. Repository name 輸入：
    你的使用者名稱.github.io
    例如：my-bakery-shop.github.io
  3. 確認選擇 Public（公開）
  4. 勾選 Add a README file
  5. 點擊 Create repository 完成建立
名稱必須完全一致
Repository 名稱必須是 使用者名稱.github.io，大小寫需一致，否則 GitHub Pages 無法自動啟用。

Step 3：開啟 GitHub Pages

讓您的網頁可以被全世界瀏覽

  1. 進入剛建立的 repository 頁面
  2. 點擊上方的 Settings（齒輪圖示）
  3. 左側選單點擊 Pages
  4. 在 Source 區域確認設定為：
    Branch: main / Folder: / (root)
  5. 點擊 Save，等待 1-2 分鐘後即可透過
    https://你的名稱.github.io 瀏覽
邀請我們協作
完成以上步驟後，請將您的 GitHub 使用者名稱 告知我們。我們會透過 collaborator 邀請的方式，將您的網頁檔案部署到此 repository。

Step 4：接受 Collaborator 邀請

我們會發送邀請，您只需接受

  1. 告知我們您的 GitHub 使用者名稱後，您會收到一封邀請 Email
  2. 點擊 Email 中的 View invitation 連結
  3. 在 GitHub 頁面點擊 Accept invitation
  4. 接受後，我們即可將網頁檔案部署到您的 repository。
    部署完成後，您的網頁就正式上線了！
部署完成後
您可以隨時在 repository 的 Settings → Collaborators 中移除我們的權限，網頁依然會正常運作。

Step 1：建立 Google Sheet

用您自己的 Google 帳號建立資料表

  1. 使用您的 Google 帳號登入 Google 雲端硬碟
    前往 drive.google.com
  2. 點擊左上角 + 新增 → Google 試算表 → 空白試算表
  3. 為試算表命名，例如：
    我的烘焙坊 - 訂單管理
不需要手動建立欄位
試算表的欄位和格式會由我們的系統自動建立，您只需要建立一個空白的試算表即可。

Step 2：共享試算表給我們

讓我們的系統能讀寫您的訂單資料

  1. 在試算表右上角點擊 共用（Share）按鈕
  2. 在「新增使用者和群組」欄位中，輸入我們提供的 服務帳號 Email
    我們會在確認訂單時提供此 Email
  3. 權限設定為 編輯者
  4. 點擊 傳送 完成共享
請選擇「編輯者」權限
系統需要「編輯者」權限才能自動建立欄位和寫入訂單資料。「檢視者」或「評論者」權限將無法正常運作。

Step 3：複製試算表網址

我們需要網址來連結您的系統

  1. 開啟您剛建立的試算表
  2. 複製瀏覽器網址列中的完整網址
    格式類似：https://docs.google.com/spreadsheets/d/xxxxx/edit
  3. 將網址傳送給我們
我們只需要網址中的 ID
試算表網址中間的一長串英數字就是 Sheet ID，我們會用它來連結您的網頁後台。您不需要自己擷取，只要貼上完整網址即可。

Step 4：等待系統設定

我們收到資訊後會完成最後設定

  1. 我們收到您的試算表網址後，會自動建立所需的欄位和分頁
  2. 同時將您的網頁部署到 GitHub Pages
  3. 全部完成後，我們會通知您，並附上網頁網址和使用說明
資料完全在您的帳號中
所有訂單、會員、商品資料都存在您自己的 Google Sheet。您可以隨時查看、匯出、備份，完全由您掌控。

設定完成確認清單

請確認以下項目都已完成

GitHub 部分

  • 已建立 GitHub 帳號
  • 已建立 使用者名稱.github.io repository
  • 已確認 Repository 為 Public
  • 已在 Settings → Pages 開啟 GitHub Pages
  • 已將 GitHub 使用者名稱告知我們
  • 已接受 Collaborator 邀請

Google Sheet 部分

  • 已用自己的 Google 帳號建立空白試算表
  • 已將試算表共享給我們的服務帳號（編輯者權限）
  • 已將試算表網址傳送給我們

需要提供給我們的資訊

1. GitHub 使用者名稱
例如：my-bakery-shop

2. Google Sheet 網址
例如：https://docs.google.com/spreadsheets/d/xxxxx/edit
傳送方式
您可以透過接單時留下的 Email 或聯絡方式，將以上資訊傳送給我們。

常見問題

GitHub 免費帳號真的夠用嗎？
完全夠用。GitHub 免費方案已包含 GitHub Pages 功能，支援自訂網域，而且沒有流量限制（合理使用範圍內）。一頁式網頁所需的儲存空間非常小，免費方案綽綽有餘。
Google Sheet 共享給你們安全嗎？
非常安全。我們使用的是 Google 服務帳號（Service Account），只能存取您主動共享的那一份試算表，無法存取您帳號中的其他檔案。您也可以隨時在共享設定中移除我們的存取權限。
可以用我已有的 GitHub / Google 帳號嗎？
當然可以。如果您已有 GitHub 帳號，直接使用即可（跳到建立 Repository 步驟）。Google 帳號也是同理，只需要建立新的試算表並共享給我們。
日後不想用了，資料會怎樣？
所有資料都在您自己的帳號中：網頁檔案在您的 GitHub repository，訂單資料在您的 Google Sheet。即使不再使用我們的服務，這些資料依然完整屬於您。您可以自由下載、備份或轉移。
可以用自己的網域名稱嗎？
可以。GitHub Pages 支援自訂網域。如果您有自己的網域（例如 mybakery.com），可以在 GitHub Pages 設定中綁定。這是加購服務的一部分，我們會協助您完成 DNS 設定。
我完全不懂技術，能完成這些步驟嗎？
可以的！這份指南已盡量簡化步驟。如果您在任何步驟遇到困難，都可以隨時聯繫我們，我們會透過螢幕共享或通話協助您完成設定。
設定完成後多久網頁會上線？
收到您的 GitHub 使用者名稱和 Google Sheet 網址後，我們通常在 1-2 個工作天內完成部署。部署完成後會立即通知您，並附上您的網頁網址。
